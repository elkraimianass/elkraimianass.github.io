Project 1 : Privilege Management and Securing Active Directory

Conduct a comprehensive analysis of Access Control Lists (ACLs) to identify security vulnerabilities and instances of excessive privileges. This process utilizes specialized tools such as BloodHound, which maps relationships and permissions in Active Directory, ADRecon for gathering information about AD environments, and AD ACL Scanner for auditing and assessing ACL configurations. The goal is to enhance security posture by pinpointing misconfigurations and potential attack vectors.


Implementation of strict password policies and enhanced authentication measures; auditing permissions and securing authentication protocols to ensure the protection of Active Directory (AD) systems. This involves establishing guidelines for password complexity and expiration, enforcing multi-factor authentication (MFA), and regularly reviewing user permissions to mitigate risks. Additionally, securing protocols such as Kerberos and NTLM is essential to prevent unauthorized access and maintain the integrity of the AD environment .



Tools Used : BloodHound, ADRecon, AD ACL Scanner, Multi-Factor Authentication (MFA) Solutions, Password Policy Management Tools, PowerShell, Kerberos Tools, NTLM Auditing Tools, SIEM Solutions, Nessus, Active Directory Administrative Center (ADAC), Group Policy Management Console (GPMC)



Project 2 : Security Audit and Web System Penetration Testing

Performing a thorough penetration test following the VAPT (Vulnerability Assessment and Penetration Testing) methodology, leveraging industry-standard tools like Burp Suite, OWASP ZAP, and Nmap. The objective is to identify and exploit critical vulnerabilities within web applications, such as SQL injection, cross-site scripting (XSS), and other high-risk flaws. This process is conducted in compliance with the OWASP Top 10 framework, ensuring a comprehensive evaluation of security risks. The testing process involves both automated scanning and manual exploitation to validate vulnerabilities and assess the overall security posture of the system.


Preparing a comprehensive audit report that outlines the vulnerabilities identified during the penetration testing process. The report includes a detailed analysis of each security flaw, its potential impact, and the risk level associated with it. Additionally, it provides strategic recommendations tailored to the application's environment, offering both immediate fixes and long-term solutions to mitigate the risks. The report also includes best practices for strengthening the overall security posture, such as improving authentication mechanisms, securing data transmission, and addressing coding vulnerabilities.



Tools Used : Burp Suite, OWASP ZAP, Nmap, Metasploit, Nessus, Acunetix, Wireshark, SQLMap, Netcat, Aircrack-ng



Project 3 : Strengthening the security of a mobile financial application and a password manager: static and dynamic analysis.

Evaluation of security mechanisms in the source code, emphasizing critical aspects such as key generation processes, encryption techniques (AES, RSA), and the validation of SSL/TLS certificates to ensure secure communication. The analysis employs a variety of tools: Jadx-GUI is used for decompiling and exploring Android application bytecode, Apk-tool facilitates the extraction and rebuilding of APK files for in-depth examination, and Ghidra serves as a powerful reverse engineering tool to analyze and scrutinize binary code, allowing for a thorough assessment of potential vulnerabilities and the overall security posture of the application. This evaluation aims to identify weaknesses in the implementation of security mechanisms and provide recommendations for enhancement.


Conducting real-time dynamic analyses using Burp Suite and Frida, which includes techniques for bypassing SSL Pinning and executing Man-in-the-Middle (MitM) attacks to test the security of data flows. This process involves intercepting and inspecting network traffic to identify potential vulnerabilities, as well as modifying application behavior at runtime to ensure that security measures are robust against various attack vectors. The analysis aims to assess the application's resilience against common threats and to provide actionable insights for strengthening data protection mechanisms.



Tools Used : Jadx-GUI, Apktool, Ghidra, Burp Suite, Frida, Wireshark, Mitmproxy, Charles Proxy, Radare2, MobSF, SSLyze, OWASP Dependency-Check

Project 5 : Risk Assessment and Security Audit : SOC (Security Operations Center) Approach to Protect Information Systems

The exploitation of vulnerabilities on Linux using Metasploit, along with the use of tools such as Nmap, Netcat, and Wireshark, aimed to explore hosts, analyze available services, and identify network vulnerabilities. This process involved scanning networks to discover active devices with Nmap, establishing connections and testing services with Netcat, and capturing and analyzing network traffic with Wireshark. By leveraging these tools, the objective was to uncover security weaknesses and assess the overall security posture of the Linux systems in question.


Implementation of a Netfilter Firewall to Secure Network Segments Using Iptables Rules and Deployment of a Snort IDS for Real-Time Network Traffic Analysis, Including the Creation of Custom Rules to Detect Attacks. This initiative involved configuring a Netfilter firewall to establish robust security measures across different network segments through the use of Iptables rules, effectively controlling incoming and outgoing traffic based on defined security policies. Additionally, deploying Snort as an Intrusion Detection System (IDS) enabled continuous monitoring and analysis of network traffic in real time. Custom rules were created to tailor detection capabilities, allowing for the identification and alerting of potential attacks or malicious activities within the network, thereby enhancing the overall security posture.



Tools Used : Metasploit, Nmap, Netcat, Wireshark, Iptables, Snort, TCPdump, OpenVAS, Aircrack-ng, Suricata, Nessus, Fail2ban



Project 6 :Strategies for the development of a SOC (Security Operations Center) in service mode

The goal is to provide a SOC as a service, offering advanced threat monitoring, effective incident detection and proactive management of responses to cyber attacks.


Comparative Study of SIEM and SOAR Solutions: An in-depth analysis was carried out to compare the SIEM solution "Elastic Stack" with the SOAR solution "Cortex XSOAR", and. These tools were evaluated to select the best options for the specific needs of a SOC.


A flexible architecture was designed, including log collection, use of VPN tunnels to secure connections and integration of orchestration tools to automate incident responses.


Deployment and Integration of Tools ; Elasticsearch and Cortex XSOAR technologies have been integrated to provide real-time monitoring and automated response to incidents. Playbooks have been developed to address specific scenarios, such as phishing and supply chain compromise.


Tools Used : Elastic Stack (ELK), Cortex XSOAR, VPN



Project 7 :Threat Detection and Response Implementation of SIEM/XDR with WAZUH

This project aimed to design and deploy an open-source SIEM/XDR solution with Wazuh to enhance the security of information systems. Emphasis was placed on improving threat detection and real-time incident response capabilities through the customization of detection rules and the implementation of active responses.


The deployed environment allowed for the collection, correlation and analysis of security events from agents installed on Linux and Windows systems. Various attacks, such as SQL injection, privilege escalation and brute force attacks were simulated to test detection capabilities. Advanced features included file integrity monitoring (FIM), integration with VirusTotal for malware detection, and active response, allowing malicious IP addresses to be blocked or suspicious files automatically deleted.


Tools Used : Wazuh, ELK Stack (Elasticsearch, Logstash, Kibana), VirusTotal, Nmap, Hydra, VirtualBox
